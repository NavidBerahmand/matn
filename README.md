# مَتن — کتابخانهٔ استاتیک کتاب‌های PDF فارسی

سایت استاتیک و کاملاً client-side برای نمایش و دانلود کتاب‌های PDF فارسی.
بدون بک‌اند و دیتابیس؛ مناسب میزبانی روی **GitHub Pages**.

## ساختار

```
index.html          صفحهٔ گالری کتاب‌ها
admin.html          ابزار محلی برای ساختن JSON یک کتاب (روی Pages لازم نیست)
books.json          «دیتابیس» سایت — فهرست کتاب‌ها
assets/style.css    استایل
assets/app.js        منطق سایت (خواندن JSON، گرید، modal، دانلود، تم)
fonts/              فونت‌های Abar (Regular / Bold / Black)
books/<slug>/       فایل‌های هر کتاب (PDF روشن، PDF تاریک، کاور)
```

## افزودن کتاب جدید

۱. یک پوشه با اسلاگ لاتین بساز: `books/<slug>/` (مثلاً `books/jennameh/`).

۲. سه فایل را داخلش بگذار:
   - نسخهٔ روشن: `<Name>-Light.pdf`
   - نسخهٔ تاریک: `<Name>-Dark.pdf`
   - کاور: `<Name>-Cover.jpg` (اختیاری — اگر نباشد placeholder خاکستری نشان داده می‌شود)

۳. فایل `admin.html` را در مرورگر باز کن (دابل‌کلیک کافی است)، فرم را پر کن،
   روی **«کپی JSON»** بزن و آبجکت را داخل آرایهٔ `books.json` پیست کن.
   (دستی هم می‌شود نوشت — فقط ساختار فیلدها را رعایت کن.)

۴. commit و push کن. GitHub Pages به‌صورت خودکار به‌روزرسانی می‌شود.

## ساخت خودکار `books.json` (GitHub Action)

به‌جای ویرایش دستیِ `books.json`، می‌توانی فقط پوشهٔ کتاب را با یک `info.txt`
بسازی؛ یک GitHub Action خودش `books.json` را تولید می‌کند.

**ساختار هر پوشه** (`books/<slug>/`):

```
books/boofkoor/
  boofkoor-Light.pdf
  boofkoor-Dark.pdf
  boofkoor-Cover.jpg      (یا .png/.webp/...)
  info.txt
```

**فرمت `info.txt`** (هر خط یک «کلید: مقدار»):

```
title: بوف کور
author: صادق هدایت
year: 1315
category: داستان بلند
tags: ادبیات معاصر، رمان، کلاسیک
pages: 120
description: متن توضیح...
```

- `slug` = نام پوشه. `language` اگر نباشد پیش‌فرض «فارسی».
- `tags` با «،» یا «,» جدا می‌شود؛ فاصله trim و تگ خالی حذف می‌شود.
- `file_light`/`file_dark`/`cover` از روی نام فایل‌ها (`-Light.pdf`، `-Dark.pdf`،
  `-Cover.<img>`) پیدا می‌شوند؛ اگر نبود، خالی می‌ماند.
- `year` و `pages` اگر عددی باشند عدد می‌شوند (ارقام فارسی هم)، وگرنه رشته می‌مانند.
- پوشهٔ ناقص (بدون `info.txt` یا بدون `title`) رد می‌شود و بقیه ساخته می‌شوند.

**نکتهٔ مهم:** با فعال‌شدن این Action، **پوشه‌ها منبعِ یگانهٔ حقیقت‌اند**؛
یعنی هر بار که داخل `books/` چیزی تغییر کند، `books.json` کامل از نوِ روی پوشه‌ها
بازنویسی می‌شود. پس اگر کتابی در `books.json` هست ولی پوشه‌اش نیست، حذف خواهد شد.

محلی هم می‌توانی همین را اجرا کنی: `python3 scripts/build_books.py`

> Action فقط روی push به `main` و فقط وقتی `books/**` تغییر کند اجرا می‌شود،
> و خودِ کامیتِ `books.json` دوباره آن را trigger نمی‌کند (جزئیات در
> `.github/workflows/build-books.yml`).

### فیلدهای هر کتاب در `books.json`

| فیلد | توضیح |
|------|-------|
| `slug` | لاتین، نام پوشه. مسیر فایل‌ها از این ساخته می‌شود |
| `title` | عنوان فارسی کتاب |
| `author` | نویسنده |
| `year` | سال (می‌تواند فارسی یا لاتین باشد) |
| `language` | زبان |
| `category` | دسته‌بندی |
| `tags` | آرایه‌ای از برچسب‌ها |
| `pages` | تعداد صفحات |
| `description` | توضیح کوتاه |
| `file_light` | نام فایل PDF روشن (یا URL کامل) |
| `file_dark` | نام فایل PDF تاریک (یا URL کامل) |
| `cover` | نام فایل کاور (یا URL کامل) |

> مسیر کامل فایل به‌صورت `books/<slug>/<file>` ساخته می‌شود.
> اگر مقدار یک فیلد یک URL کامل (`http…`) باشد، همان مستقیم استفاده می‌شود
> (مثلاً وقتی PDFها را روی **GitHub Releases** می‌گذاری).

## دانلود اجباری (Light / Dark)

داخل modal دو دکمهٔ جدا برای دانلود نسخهٔ روشن و تاریک هست.
- فایل‌های هم‌دامنه (داخل `books/`) با attribute `download` مستقیم دانلود می‌شوند.
- فایل‌های cross-origin (مثل Releases): کد ابتدا فایل را با `fetch` به‌صورت blob می‌گیرد
  و بعد دانلود می‌کند تا در مرورگر باز نشود. اگر میزبان CORS را ببندد،
  به‌عنوان آخرین راه در تب جدید باز می‌شود.

## تم روشن/تاریک سایت

دکمهٔ بالای صفحه ظاهر خود سایت را بین روشن و تاریک سوییچ می‌کند و انتخاب در
`localStorage` ذخیره می‌شود. (این مستقل از نسخهٔ PDF روشن/تاریک است.)

## اجرای محلی

به‌خاطر `fetch` روی `books.json`، فایل را با یک سرور ساده باز کن (نه `file://`):

```bash
python3 -m http.server 8000
# سپس: http://localhost:8000
```
